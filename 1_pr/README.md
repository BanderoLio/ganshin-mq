## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ pnpm install
```

## Compile and run the project

```bash
# development
$ pnpm run start

# watch mode
$ pnpm run start:dev

# production mode
$ pnpm run start:prod
```

## Run tests

```bash
# unit tests
$ pnpm run test

# test coverage
$ pnpm run test:cov
```

### E2E tests

E2E tests use a real PostgreSQL database. Before the suite runs, [test/global-setup.cjs](test/global-setup.cjs) applies migrations (`prisma migrate deploy`). The environment variable **`DATABASE_URL` is required**; if it is missing, the global setup fails.

1. Start PostgreSQL and create a database (or reuse an empty one). The URL must match your instance, for example:

   `postgresql://USER:PASSWORD@HOST:PORT/DATABASE`

2. From the project directory (`1_pr`), run:

   ```bash
   export DATABASE_URL="postgresql://app:app@localhost:5432/store"
   pnpm run test:e2e
   ```

   On one line:

   ```bash
   DATABASE_URL="postgresql://app:app@localhost:5432/store" pnpm run test:e2e
   ```

3. Optional: copy [.env.test.example](.env.test.example) to `.env.test`, set `DATABASE_URL` there, then load it before tests (Nest and Prisma also read `.env` if present):

   ```bash
   set -a && source .env.test && set +a && pnpm run test:e2e
   ```

**Using Docker Compose (only Postgres).** If you run `docker compose up -d postgres` from this folder, the default credentials in [docker-compose.yml](docker-compose.yml) are `app` / `app`, database `store`, port `5432` on the host. Then:

```bash
export DATABASE_URL="postgresql://app:app@localhost:5432/store"
pnpm run test:e2e
```

The e2e suite truncates data between cases; use a dedicated database, not production.

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ pnpm install -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.
